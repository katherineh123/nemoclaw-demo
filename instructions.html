Background

What is NemoClaw?

Agent Harnesses

An agent harness is a framework that lets an AI model do more than just answer questions — it gives the AI the ability to take actions: read files, run code, browse the web, call external tools, and chain these steps together to complete a goal. Think of it as scaffolding that turns a smart assistant into an autonomous worker.

OpenClaw

OpenClaw is an open-source agent harness — a community-built platform for running AI agents. It defines how agents access tools, manage memory, and handle multi-step tasks. Many organizations use OpenClaw as the foundation for their own AI agent systems.

NemoClaw

NemoClaw is NVIDIA's implementation of the OpenClaw framework. It's built on the same open-source foundation but optimized for NVIDIA infrastructure, integrated with NVIDIA's model stack, and extended with enterprise features like data connectors, security controls, and deployment tooling. What you're using today is NemoClaw running on a VM, with Claude (Anthropic's AI model) powering the reasoning underneath.

Data Access

How NemoClaw connects to your data

NemoClaw can pull data from a wide range of sources — not just files on the VM, but cloud drives, databases, code repositories, and more. It does this through data connectors: command-line tools (CLIs) that the agent can use to reach out and retrieve data just like a developer would.

🐙 GitHub CLI — code & data repos
📁 Google Drive CLI — shared files & spreadsheets
📂 git clone — version-controlled datasets
🗄️ Database connectors — SQL, data warehouses
☁️ Cloud storage — S3, Azure Blob, GCS

For today's demo, factory data is stored in a GitHub repository. When you reference it in your prompt, NemoClaw uses git clone to pull the data directly into its working environment — no manual downloads or uploads needed.

The Data

Factory delivery & headcount data

The dataset covers 24 months (May 2024 – April 2026) of monthly headcount and on-time delivery performance for two factories. Each row is one factory, one month.

Column What it means
monthThe month of the observation (YYYY-MM-DD)
factoryWhich factory: Factory A (Europe) or Factory B (Asia)
total_headcountTotal workers on site that month
new_hiresWorkers who joined this month (not yet productive)
workers_1moWorkers who joined last month (partially ramped)
workers_2moWorkers who joined two months ago (almost fully ramped)
tenuredWorkers with 3+ months on site (fully productive)
fulfillment_pct% of customer orders delivered on time that month

Here's a sample of what the data looks like:

month,factory,total_headcount,new_hires,workers_1mo,workers_2mo,tenured,fulfillment_pct 2024-05-01,Factory A (Europe),249,9,0,8,232,60.67 2024-06-01,Factory A (Europe),247,10,9,0,228,60.99 2024-07-01,Factory A (Europe),242,6,10,9,217,57.54 2024-08-01,Factory A (Europe),271,42,6,10,213,57.32 2024-09-01,Factory A (Europe),265,6,42,6,211,58.66

Notice how August had a big hiring surge (42 new hires) — but fulfillment didn't improve immediately, because those workers need time to ramp up. That's exactly the kind of pattern NemoClaw will find on its own.

Prompt 1 — Paste this into NemoClaw
i have some data about how worker headcount affected factory delivery timeliness over the past couple years for my germany factory — it's in https://github.com/katherineh123/nemoclaw-demo.git, file is called factory-delivery-data. theres also a knowledge base there that gives you domain knowledge/context about our company, make sure to reference that. based on my most recent headcount numbers, whats the projected timeliness for next month? im trying to figure out if i need to add additional headcount or not. we need to be at at least 70% fulfillment.

What to expect: NemoClaw will pull the data from GitHub, explore the headcount columns, discover how worker tenure affects productivity, fit a forecasting model, and give you a projection with a plain-language recommendation — all in one response.
