Getting Started with NemoClaw

Setup

What you're working with

You're interacting with NemoClaw — an AI agent that can read data, write code, run analyses, and publish results. It's hosted on a virtual machine (VM). Throughout this tutorial, the agent sets itself up autonomously — locating your data, connecting to it securely, and acquiring whatever tools it needs. You don't configure any of that; you just describe what you want.

You communicate with NemoClaw by typing plain-English requests — just like a message you'd send to a colleague. NemoClaw figures out the technical steps, does the work, and reports back.

Background

What is NemoClaw?

Agent Harnesses

An agent harness is a framework that lets an AI model do more than just answer questions — it gives the AI the ability to take actions: read files, run code, browse the web, call external tools, and chain these steps together to complete a goal. Think of it as scaffolding that turns a smart assistant into an autonomous worker.

OpenClaw

OpenClaw is an open-source agent harness — a community-built platform for running AI agents. It defines how agents access tools, manage memory, and handle multi-step tasks. Many organizations use OpenClaw as the foundation for their own AI agent systems.

NemoClaw

NemoClaw is NVIDIA's implementation of the OpenClaw framework. It's built on the same open-source foundation but optimized for NVIDIA infrastructure, integrated with NVIDIA's model stack, and extended with enterprise features like data connectors, security controls, and deployment tooling. What you're using today is NemoClaw running on a VM, with Claude (Anthropic's AI model) powering the reasoning underneath.

Data Access

How NemoClaw connects to your data

NemoClaw can pull data from a wide range of sources — not just files on the VM, but cloud drives, databases, code repositories, and more. It does this through data connectors: command-line tools (CLIs) that the agent can use to reach out and retrieve data just like a developer would.

For today's demo, factory data is stored in a GitHub repository. When you reference it in your prompt, NemoClaw uses Github's CLI (git clone) to pull the data directly into its working environment. It then writes updated content directly back to Github so you can see changes in real time— no manual downloads or uploads needed.

Domain Knowledge

How the agent learns about your company

Company documentation

AI models are general-purpose — they don't know your factories, your KPIs, or how you define on-time delivery. To give the agent that context, you simply point it to documentation and ask it to read before it starts, just like a new hire reads an onboarding wiki before their first project. Unlike a new hire who might click through pages in a browser, your agent will access it directly via CLI — pulling the file from wherever it lives and reading it in full before proceeding.

Skills

For more structured guidance, you can write a skill — a special document written for the agent rather than for humans. A skill is a plain-language, step-by-step instruction file (e.g. skill.md) that tells the agent exactly how to approach a specific type of task: what data to use, what model to fit, how to format results, where to publish. The agent loads it like a briefing document before starting work.

The enterprise-knowledge-base.md file in this repo is an example: it describes the factories, defines the performance thresholds, specifies the forecasting methodology, and tells the agent where to publish charts — all in plain English.

Your turn

Try your first prompt

Scenario

You are an operations manager at a manufacturing company that owns two factories — one in Germany, one in China. You need to ensure that next month's on-time fulfillment rate at the German factory does not fall below 70%. To hit that target, you can hire additional workers. But while it's intuitive that more workers means better fulfillment, it's unclear exactly what the relationship is between the number of new hires and your actual fulfillment rate. Fortunately, you do have historical data: monthly new hire counts and monthly fulfillment rates for the past two years. So maybe you and your agent can figure something out.

Open your NemoClaw session and type the prompt below exactly as written. NemoClaw will locate the data, analyze it, write a mathematical formula that predicts future factory output, and tell you where Germany's on-time delivery rate is headed. All you have to do is describe the outcome you want to reach — due to its exceptional autonomy, the agent figures out the code and technicalities in between.

Prompt 1
I have some data about how worker headcount affected factory delivery timeliness over the past couple years for my Germany factory — it's in https://github.com/katherineh123/nemoclaw-demo.git, file is called factory-delivery-data. There's also a knowledge base there that gives you domain knowledge/context about our company — make sure to read the full thing before proceeding. Based on my most recent headcount numbers, what's the projected timeliness for next month? I'm trying to figure out if I need to add additional headcount or not. We need to be at at least 70% fulfillment.
Prompt 2
Can you show me this as a visualization?
Prompt 3
I need to reach a higher fulfillment rate in my Germany factory and that's not gonna happen based on your projection. What if I move 50 people from my Asia factory to Germany? Can you give a visualization showing what both factories look like?
Prompt 4
Can you make it interactive so I can change how much headcount I move between the 2 factories and see how it affects fulfillment for both?
Security

How OpenShell makes this safe to try

NemoClaw is designed to take action, not just answer questions. OpenShell is the safety layer around that autonomy: it gives each agent a controlled workspace and limits what the agent can access.

The agent can use credentials without seeing them

In this demo, NemoClaw needs access to services such as GitHub and the model endpoint. OpenShell handles those credentials behind the scenes. The agent can complete approved actions, like pulling data or publishing a chart, but it is not given the actual API keys to read, print, or save.

The agent only reaches approved places

OpenShell also controls where the agent is allowed to connect. For example, a demo can allow access to GitHub for the dataset and dashboard, while blocking unrelated websites or internal systems. If the agent tries to go somewhere outside the allowed list, OpenShell stops the request.

This means the agent can still work independently, but within boundaries your organization defines.

How today's demo is set up

For this hands-on session, everything runs from one virtual machine. That VM hosts multiple OpenShell sandboxes, and each sandbox contains its own NemoClaw session. Each participant gets a separate link into their own sandbox, so everyone can try the same workflow independently without sharing a chat history or workspace.
